PLAYSCORE - GAME REVIEW PLATFORM
PROJECT REPORT

================================================================================
1. DESCRIPTION OF SYSTEM
================================================================================

PlayScore is a professional game review platform built with Laravel 11 that allows 
administrators to create and manage game reviews while providing users with a rich 
browsing experience enhanced by AI-powered recommendations.

Core Features:
- Game review management with CRUD operations
- Admin dashboard for content control
- User authentication (login/registration)
- Search and filter functionality
- Pagination for efficient data display
- Bilingual support (English/Arabic)
- AI chatbot for personalized game recommendations
- RAWG API integration for automatic game metadata fetching
- Server-side and client-side caching for performance
- Responsive dark theme design
- Skeleton loading placeholders for better UX during content loading
- User comments system on game reviews with authentication

The system serves two main user types:
1. Administrators: Can add, edit, and delete game reviews via dashboard
2. Regular Users: Can browse reviews, search games, and interact with AI assistant

Current Status: 14 professionally written game reviews covering popular titles
including Elden Ring, Cyberpunk 2077, The Witcher 3, and more.

2. AUTOMATION PROCESS AND WORKFLOW
================================================================================

User Workflow:
1. Visit homepage â†’ Browse game reviews (Popular/Recent tabs)
2. Use search bar to find specific games
3. Register/Login for personalized features
4. Click on game to view detailed review
5. Read and post comments on game reviews (authentication required)
6. Click AI chatbot to get game recommendations
7. Switch between English and Arabic languages seamlessly

Admin Workflow:
1. Login to admin dashboard (/admin/dashboard)
2. Enter RAWG ID of game to review
3. System automatically fetches game title, description, and images from RAWG API
4. Admin writes rating (0-100) and review text
5. Submit to database
6. Cache automatically cleared to show new review immediately
7. Edit or delete reviews as needed with automatic cache invalidation

Automated Processes:
- Game metadata fetching: When admin enters RAWG ID, system automatically 
  retrieves game title, description, release date, and cover image
- Caching system: Popular/recent game lists cached for 5 minutes, RAWG API 
  responses cached for 1 hour
- Cache invalidation: Automatically flushes cache when games are 
  created/updated/deleted
- Language switching: Changes interface language without page reload
- Image lazy loading: Images load on-demand as user scrolls
- AI conversation context: Full chat history sent with each message for 
  context-aware responses

================================================================================
3. TECHNOLOGY STACK CHOICES
================================================================================

Backend Framework: Laravel 11 (PHP 8.2+)
Why: Industry-standard MVC framework with excellent ORM (Eloquent), built-in 
authentication, routing, and caching. Rapid development with robust security features.

Database: MySQL 8.4
Why: Reliable relational database with excellent Laravel integration. Handles 
relationships between users and games efficiently.

Frontend Framework: Bootstrap 5
Why: Fast responsive design, pre-built components, excellent documentation. 
Reduces development time while maintaining professional appearance.

JavaScript: Vanilla JS (with minimal jQuery for dashboard AJAX)
Why: Better performance, no unnecessary dependencies, modern async/await support 
for API calls.

External APIs:
- RAWG API: Comprehensive game database with metadata, images, and ratings. 
  Saves time vs manual data entry.
- Google Gemini AI (gemini-2.0-flash): Advanced language model for natural 
  conversations about games. Fast response times and accurate recommendations.

Performance Tools:
- Laravel Cache (file-based): Simple setup, no Redis needed for development. 
  Significantly improves page load times.
- Lazy Loading: Reduces initial page weight, faster first contentful paint.

Development Environment:
- Laragon: All-in-one local server (Apache, MySQL, PHP) with easy configuration.
- Git/GitHub: Version control for collaboration and deployment.
- Composer: PHP dependency management.
- NPM: Frontend asset compilation.

Design Choices:
- Dark theme: Modern aesthetic, reduces eye strain, popular in gaming community.
- Bilingual: Expands audience to Arabic-speaking gamers (large underserved market).
- Responsive: Works on mobile, tablet, and desktop devices.

B. CHALLENGES AND SOLUTIONS

Challenge 1: RAWG API Rate Limiting and Timeouts
Problem: RAWG API sometimes slow or hits rate limits, causing delays in 
fetching game data.
Solution: Implemented 1-hour caching for RAWG API responses with 5-second 
timeout. Subsequent requests for same game served instantly from cache.

Challenge 2: Pagination Showing Incorrect Counts
Problem: Dashboard showed "Showing 1-10 of 10 reviews" when there were 14 
reviews in database.
Solution: Changed API call from default limit=10 to limit=1000 to fetch all 
reviews, then implement client-side pagination for accurate counts.

Challenge 3: Dark Theme Bootstrap Overrides
Problem: Bootstrap default styles (white backgrounds, dark text) overriding 
custom dark theme CSS.
Solution: Used high-specificity selectors (#paginationButtons .page-item) and 
!important flags to override Bootstrap defaults. Created separate dashboard.css 
file for better organization.

Challenge 4: Chatbot No Conversation Memory
Problem: AI chatbot treated each message independently, couldn't answer 
follow-up questions like "tell me more about that".
Solution: Store conversation history in JavaScript array, send full message 
history with each API request. Backend builds proper Gemini conversation 
format with user/model roles.

Challenge 5: Alert Popups Blocking User Experience
Problem: alert() calls for login errors and registration success interrupting 
user flow, can't copy text, must click OK.
Solution: Replaced all alerts with inline Bootstrap alert messages that appear 
smoothly, auto-hide for success messages, stay visible for errors.

Challenge 6: jQuery and Vanilla JS Mixed
Problem: Homepage used vanilla JS, dashboard used jQuery, causing confusion 
and larger bundle size.
Solution: Removed jQuery from homepage (replaced $(document).ready with 
addEventListener), kept only in dashboard for AJAX convenience. Reduced 
dependencies.

Challenge 7: Inline CSS Scattered Everywhere
Problem: Inline styles throughout Blade templates making maintenance difficult, 
inconsistent styling.
Solution: Created external CSS files (dashboard.css, chat.css), extracted most 
inline styles using multi_replace_string_in_file. Some complex inline styles 
remain due to dynamic Blade content.

Challenge 8: Language Switching Not Applying to Dynamic Content
Problem: JavaScript-generated content (game cards, navigation) not translated 
when switching language.
Solution: Exposed translations to JavaScript via window.translations object in 
app.blade.php layout. JS accesses translations like window.translations.login.

Challenge 9: Cache Not Invalidating After Updates
Problem: After adding/editing/deleting games, old cached data still showing to 
users.
Solution: Added Cache::flush() calls in GameController after create(), update(), 
and delete() operations. Forces fresh data fetch on next request.

Challenge 10: Search Only Working on Homepage
Problem: Search form in navbar appeared on all pages but only functioned on 
homepage.
Solution: Created global-search.js loaded in main layout. Handles search form 
submit on all pages, redirects to /search?q=query route.

Challenge 11: mysqldump Command Not Found
Problem: Trying to export database, mysqldump not in system PATH.
Solution: Used full path to Laragon's MySQL bin directory: 
c:\laragon\bin\mysql\mysql-8.4.3-winx64\bin\mysqldump.exe

Challenge 12: Chatbot Accessible to All Users
Problem: Initially anyone could use AI assistant, wanted to restrict to 
logged-in users only.
Solution: Added auth middleware to /api/chat route, check authentication status 
in chat.js before allowing interaction. Show "Please login" message if not 
authenticated.

Challenge 13: Blank Screens During Content Loading
Problem: Users seeing blank white space for 1-2 seconds while games loaded, 
creating perception of slowness or broken page.
Solution: Implemented skeleton loading placeholders with animated gradient 
shimmer. Show 4-6 skeleton cards immediately while fetching real data. Provides 
instant visual feedback and makes page feel faster even though load time is 
identical.

Challenge 14: Authorization for Comment Deletion
Problem: Need to allow users to delete their own comments, admins to delete 
any comment, but prevent users from deleting others' comments.
Solution: Implemented authorization check in CommentController destroy() 
method. Check if Auth::id() matches comment user_id OR user is_admin. Return 
403 Forbidden if neither condition met. Frontend hides delete button for 
unauthorized users.
